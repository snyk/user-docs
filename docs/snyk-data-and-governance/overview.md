# Overview

To assist you with operational and governance decisions when using Snyk in your production environment, Snyk provides the following resources:

* [Working with Snyk](working-with-snyk.md): secure and responsible handling of user data, with a strong emphasis on privacy and compliance.
* [Regional hosting and data residency](regional-hosting-and-data-residency.md): ensures data residency requirements are met and reduces data latency.
* [How Snyk handles your data](how-snyk-handles-your-data.md): Review data handling and platform architecture.
* [How Snyk incorporates generative AI into the platform](how-snyk-incorporates-generative-ai-into-the-platform.md): Understand AI governance, including data processing, training exclusions, and controls.
* [Disclosure of a vulnerability in an open source glossary](disclosure-of-a-vulnerability-in-an-open-source-package.md): Report a vulnerability in the ecosystem Snyk tracks.
* [Reporting security issues](reporting-security-issues.md): Report a security issue in Snyk services.
* [Snyk terms of support and services glossary](snyk-terms-of-support-and-services-glossary/): Review support scope and response expectations.
* [How Snyk counts assets](how-does-snyk-count-assets.md), [What counts as a test](what-counts-as-a-test.md), and [Snyk Platform credits](snyk-platform-access-credits.md): Review these pages for usage measurement, billing alignment, and capacity planning.&#x20;
* [Snyk for Government (US)](snyk-for-government-us.md): Review FedRAMP and NIST requirements, platform differences, and feature limitations for public-sector deployments.
